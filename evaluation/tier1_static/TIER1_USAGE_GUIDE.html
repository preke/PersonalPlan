1. Tier 1 评估概览

Tier 1 是 plan 静态质量评估（pre-execution）—— 在 plan 还没真正执行前， 评估它的结构合理性、内容质量、个性化程度。共 10 项指标，分 3 类：

类别指标是否需要 LLM API耗时（305 条）
规则 SchemaValid_Pass / ToolAgent_Perm / Topo_Consist / ATR / GED_sim ❌ 不需要 ~2–3 min（CPU）
LLM judge Pers + 3 子维（SkillMatch / GoalOrient / BgAdapt）
Ped + 4 子维（PRR / NDAR / SPR / IAR）		 ✅ 需要 bianxie.ai API key ~40 min（8 workers）
反事实 PVS（规则）/ PNG（LLM judge） PVS ❌ ／ PNG ✅ PVS ~30s ／ PNG ~6 min
默认 LLM judge backend：gemini-3.1-pro-preview（via bianxie.ai 代理）。 历史脚注：早期用过 gpt-5-2025-08-07，2026-05-22 起统一切到 gemini， 长期默认不再改。

3.1 输入 plans JSONL 格式

每行一个 JSON 对象，必须包含外层 wrapper：

{
  "question_id": "1031554",
  "profile_index": 1,
  "generated_plan": {
    "input":  { "query": "...", "learner": { "about_me": "...", "top_tags": [...] } },
    "output": { "agents": [...], "subtasks": [...], "execution_order": [...] }
  }
}

外层支持以下两种别名（都会被自动 unwrap）：

  • generated_plan（baseline 输出标准）
  • plan（gold dataset 标准）

4. 10 项指标说明

#指标类型取值含义
1 SchemaValid_Pass 规则 {0, 1} schema 合规综合门：top-level 字段齐全、agent.tools ⊂ 8 工具白名单、 step.agent 已声明、step.tool 在 agent 工具列表里、execution_order = 所有 step.id 集合
2 ToolAgent_Perm 规则 [0, 1] 或 NaN 带工具的 step 中，工具 ∈ 该 agent.tools 的比例。 NaN = 该 plan 完全没用工具（不算失败）
3 Topo_Consist 规则 [0, 1] depends_on 边中「dep 早于 dependent」在 execution_order 顺序里的比例
4 ATR 规则 [0, 1] 或 NaN Agent-Tool-Subtask 三方语义对齐（embedding mode 用 MiniLM 余弦阈值 0.10）
5 GED_sim 规则（vs gold） [0, 1] 或 NaN 1 − GED / max(|S|+|O|, |S*|+|O*|)，与 gold plan 的图编辑距离相似度
6 Pers + 3 子维 LLM judge [0, 1] 个性化得分（SkillMatch / GoalOrient / BgAdapt 三子维平均）
7 Ped + 4 子维 LLM judge [0, 1] 教学法得分（PRR / NDAR / SPR / IAR 四子维平均）
8 PVS 反事实（规则） [0, 1] Profile Variation Sensitivity = 1 − mean pairwise GED-sim per qid， 然后跨 qid 取均值。**需要多 profile qid**
9 PNG 反事实（LLM） (-1, 1) Profile-Negative Gain = Pers(real plan with real profile) − Pers(other plan with this profile swapped in)
SchemaValid_Pass 的 6 个子检查（按出现顺序）：
  1. 顶层有 input 和 output
  2. input.query + input.learner.{about_me, top_tags} 都齐全
  3. output.agents 非空、每个 agent 4 字段齐全
  4. 每个 agent.tools[*] ∈ 8 工具白名单
  5. 每个 step 8 字段齐全（含 depends_on）；step.agent ∈ 已声明 agents； step.tool ∈ 该 agent 的 tools 列表
  6. flatten(execution_order) 集合 = 所有 step.id 集合
任一子检查失败 → 该 plan 得 0。

5. 推荐方式 A：tier1_run.py 一条命令

新数据集做完整 Tier 1，一条命令搞定：

cd D:/codex_project/multi_agent_datasets/Evaluation

python tier1_run.py \
    --input  <你的_plans.jsonl> \
    --output <输出.csv> \
    --gold   D:/codex_project/multi_agent_datasets/multi_agent_dataset_filtered_qap.jsonl \
    --qap    D:/codex_project/multi_agent_datasets/filtered_qap.jsonl \
    --workers 8 \
    --pvs --png

5.1 完整参数表

参数默认值说明
--input必填plans JSONL 路径
--output必填输出 CSV 路径
--goldEvaluation/multi_agent_dataset_filtered_qap_v15_goodplus.jsonl GED-sim 和 PVS 用的 gold；推荐用顶层 3043 全量
--qapEvaluation/filtered_qap.jsonl NDAR 用的 accepted-answer 字典
--judgegemini-3.1-pro-preview LLM judge backend，long-term 不改
--workers4LLM 并发数；全量跑建议 8
--rule-onlyFalse只跑 5 项规则，无需 API key（快速 smoke）
--pvsFalse额外算 PVS 聚合
--pngFalse额外算 PNG 聚合（需 API key）
--atr-modeembedding embedding（需 sentence-transformers）或 keyword
--atr-threshold0.10 embedding 模式的余弦相似度阈值
--limitNone只跑前 N 条（smoke 用）

5.2 想只跑某一类指标

# 只跑 5 项规则（无需 API key，~3 min）
python tier1_run.py --input plans.jsonl --output out.csv --rule-only

# 跑 rule + judge（不要反事实）
python tier1_run.py --input plans.jsonl --output out.csv

# 跑全部 10 项（含 PVS + PNG）
python tier1_run.py --input plans.jsonl --output out.csv --pvs --png

8. 输出格式

8.1 主 CSV 字段

question_id, profile_index,
SchemaValid_Pass, ToolAgent_Perm, Topo_Consist, ATR, GED_sim,        # 5 项规则
Pers, Pers_SkillMatch, Pers_GoalOrient, Pers_BgAdapt,                 # Pers + 3 子维
Ped,  Ped_PRR, Ped_NDAR, Ped_SPR, Ped_IAR,                            # Ped + 4 子维
error                                                                 # 异常信息

8.2 stdout 打印

  • 每项指标的均值 + n（非 NaN 数）
  • 有 error 的行数
  • PVS 聚合（如果加了 --pvs）
  • PNG 聚合 + paired-t p-value（如果加了 --png）

8.3 PVS / PNG 单独 CSV

  • pvs_<NAME>.csv — per-qid PVS + n_profiles
  • pvs_summary.csv — 跨 baseline 汇总（mean/median/min/max/std）
  • png_<NAME>.csv — per-pair PNG = Pers_real − Pers_shuf_cf

9. 常见踩坑

坑 1：GED_sim 全 NaN
原因：--gold 用了只覆盖 test split 的 gold_v15_test305.jsonl，但你 plan 用的是 dev 或 train qid。
修法：--gold 一律用顶层全量 multi_agent_dataset_filtered_qap.jsonl（3043 条），覆盖所有 split。
坑 2：PVS = 0 或 "multi-profile qids: 0"
原因：参考脚本里写死了 gpt-5 qid 做 universe filter； 如果新数据用 dev qid，过滤光了。
修法：照 _run_pvs_grpo_v4.py 用文件自己的 qid，不外加过滤。
坑 3：ModuleNotFoundError: sentence_transformers
原因：ATR embedding mode 需要这个包，本机没装。
修法：两选一——
  1. pip install sentence-transformers
  2. 加 --atr-mode keyword 走纯关键词模式（精度略低但能跑）
坑 4：API key 报错 / "missing OPENAI_PROXY_API_KEY"
修法：检查 D:/codex_project/multi_agent_datasets/.env 里有 OPENAI_PROXY_API_KEY=...，或环境变量里有。
坑 5：plan JSON 格式坏（KeyError / ValidationError）
修法：先跑 --rule-only，看 error 列是否报 schema 异常。如果是 LLM 输出的 JSON 本身坏，参考 baselines/T1_results/_convert_grpo_v4.py 用 json_repair 包兜底修。
坑 6：bianxie.ai 限速 / Claude 偶发 schema bug
修法：判 judge 时 --workers 4 起步，遇 429 自动 retry；Claude 路径偶发空 JSON 反复跑 1-2 次基本能成。
坑 7：续跑 / done set 跳过想重跑的行
原因：_run_* 脚本不带 resume； tier1_run.py 也是从头跑。要部分重跑必须先删旧 CSV 或换 --output 路径。
