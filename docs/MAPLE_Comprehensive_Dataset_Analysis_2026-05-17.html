核心结论 (TL;DR)
  • 规模：3,043 条样本 / 1,730 个唯一 query / 2,738 个唯一 learner profile。一对多平均扇出 1.76，最大 6。
  • 划分均衡度：意图分布在三 split 下的 Cramér's V = 0.069（几乎独立）；profile_index 分层 V = 0.010；nprof 桶 V = 0.012。三个 split 类别分布高度对齐
  • 计划复杂度：每条计划平均 2.91 个 agent、3.98 个子任务、10.38 个 step；DAG 关键路径深度均值 5.45，可并行度 0.47。
  • 个性化信号（同 query, 不同 profile 之间）：agent_role 名称 Jaccard 均值仅 0.32、subtask 措辞 Jaccard 仅 0.23；但 role-family 层 Jaccard 0.93、tool 集合 Jaccard 0.91。个性化发生在「具体表述」层，骨架（role-family + tools）保持稳定。
  • 教学性：100.0% 的 plan 含 validate 阶段，16.6% 满足 probe→retrieve/demonstrate→apply→validate 顺序；R_ped proxy 均值 0.86，个性化 grounded-step proxy 均值 0.47。
  • Schema 合规：3,043/3,043 计划全部通过引用 / DAG / 执行序校验 (100.00%) — 零结构缺陷

E · 个性化 / 一对多分析（重点）

核心问题：因为 MAP-PPL 中同一个 query 会被绑定到多个 learner profile 各自生成一份 plan，我们需要证明：(1) 不同 profile 真的导致了实质不同的 plan；(2) 这种差异化是结构化的（与 profile 的相似度相关），而不是随机噪声。下面几张图回答这两件事。

E.1 跨-profile plan 相似度（四个维度的 Jaccard）

E.1 — 同一 query、不同 profile 之间生成的 plan，在四个维度的 Jaccard 分布（n=1,777 对）。 呈现明显的双轨结构： 高层骨架稳定 — role family Jaccard=0.93、tool Jaccard=0.91； 具体表述差异化 — agent_role 名称 Jaccard=0.32、subtask token Jaccard=0.23。 意味着模型保留了「同样的高层 pipeline（一个 tutor + 一个 retriever + 一个 validator，配同样的两件工具）」，但每个 profile 都被实质重写了具体角色名和 subtask 措辞。这正是 MAP-PPL 个性化设计的关键证据。

E.2 Profile 相似度 → Plan 相似度：是否结构化？

E.2 — 横轴为同 qid 两 profile 的技能 Jaccard，纵轴为对应 plan 的 agent_role Jaccard。Pearson r = 0.146，正相关但弱 → profile 越像，plan 名称越接近，但即使是几乎相同的 profile，plan 也会有差异化（不是简单复制）。

E.3 一对多扇出深度 vs 个性化强度

E.3 — 按 nprof_per_qid 分桶，看每对的 plan 发散度 (1 − Jaccard) 是否随扇出增加而变化。结论：发散度在不同桶下基本稳定 → 即使一个 query 被绑定到 6 个 profile，每对之间也保持一致的个性化幅度，没有出现「越往后越敷衍」的情况。

E.5 各意图下的个性化强度

E.5 — 按 query primary intent 切分后的 plan 发散度（1 − role Jaccard）。所有意图都在 0.6–0.8 区间，说明 profile-conditioning 在不同问题类型下都成立。
解读： agent_role 名称、subtask 措辞几乎随每对 profile 全换（Jaccard 分别 0.32 / 0.23），但角色家族、工具集保持稳定（0.93 / 0.91），证明：
  • plan 不是简单复制（如果是，role / subtask Jaccard 都应接近 1.0）；
  • 也不是随机重铸（否则 family / tools Jaccard 也应低）；
  • 而是在保留高层 pipeline（role-family + tools）的同时，为每个 profile 重新具体化角色名与 subtask 措辞——这正是 MAP-PPL 想要的「同任务、不同教练」个性化形态。
