1. 设计原理

1.1 什么是 Hypervolume，以及为什么"缺哪项就提哪项"

把三个 reward 当成三维空间里的坐标。每个 batch 跑完后， (R_struct_mean, R_pers_mean, R_ped_mean) 是一个点。

Hypervolume (HV) = 这个点和一个固定参考点 r（你希望"至少达到"的下限，比如 r = (0.3, 0.3, 0.3)） 构成的长方体体积。HV 越大 = 三项 reward 越远离参考点 = Pareto 状态越好。

HV 对每一维的偏导数 ∂HV/∂R_i = ∏_{j≠i} (R_j - r_j) —— 也就是说， HV 关于"目前最低那一维"的梯度最大。所以最快增大 HV 的方法 = 把权重往最低的那一维倾斜。

1.2 简化实现：指数化乘法更新

不要去做完整的 HV 梯度计算（数值不稳、容易爆）。用一个等价的启发式：

gap_i = target_i - EMA(R_i)            # 正 = 这一维拖后腿
w_i ← w_i · exp(lr · gap_i)            # 拖后腿就指数级涨权重
w ← clip(w, w_floor, w_cap)
w ← w / sum(w)                          # 归一化保持预算总和 = 1

三个关键参数：

  • target：每一维的"期望水位"，参考点。比如 [0.5, 0.4, 0.4]。
  • lr（weight_lr）：权重学习率。建议 0.3 ~ 0.5，太大会震荡。
  • update_every：多少 reward fn 调用更新一次。建议 50 ~ 200。

1.3 为什么用 EMA 而不是瞬时均值

单个 batch 的 raw mean 噪声大（G=8 个 rollout 而已），直接拿来更新权重会让 w 跳来跳去。 用 EMA 平滑：EMA ← (1-α)·EMA + α·this_batch，α = 0.1 大概等价 10 个 batch 的滑动平均。

1.4 和 z-score / GDPO 的关系

关键：动态权重读的是 RAW reward 均值，不是 z-score 后的。
z-score 之后的均值永远是 0，看不出哪一维"拖后腿"。所以 Reweighter 必须在 z-score 之前 观察每一维的绝对水位，然后输出新的权重；这个权重再乘到 z-score 之后 的值上。

3. Step 1 在 rewards.py 文件顶部加 Reweighter 类

插入位置：rewards.py 第 53 行附近（from plan_utils import ... 之后， # R_struct 注释之前）。

GRPO/rewards.py — 新增（约 line 55 开始）
import numpy as np
from collections import defaultdict, deque
from datetime import datetime


class HypervolumeReweighter:
    """
    简化版 hypervolume-guided 动态权重更新（参考 arxiv 2509.11452）。

    每 `update_every` 次 reward_fn 调用，根据各 component 的 EMA 均值
    与 target reference 的差距，按指数化乘法更新权重。
    """

    def __init__(
        self,
        component_names: tuple[str, ...] = ("struct", "pers", "ped"),
        init_weights: tuple[float, ...] = (0.55, 0.25, 0.20),
        reference: tuple[float, ...] = (0.50, 0.40, 0.40),
        update_every: int = 100,
        ema_alpha: float = 0.10,
        weight_lr: float = 0.3,
        w_floor: float = 0.05,
        w_cap: float = 0.70,
        log_path: str | None = None,
        warmup_steps: int = 50,
    ):
        assert len(component_names) == len(init_weights) == len(reference)
        self.names = component_names
        self.weights = np.array(init_weights, dtype=float)
        self.ref = np.array(reference, dtype=float)
        self.update_every = update_every
        self.ema_alpha = ema_alpha
        self.weight_lr = weight_lr
        self.w_floor = w_floor
        self.w_cap = w_cap
        self.warmup_steps = warmup_steps  # 不更新权重的前 N 步
        self.ema: np.ndarray | None = None
        self.step = 0
        self.log_path = log_path
        if log_path:
            with open(log_path, "w") as f:
                f.write("# step\ttimestamp\tema\tweights\n")

    def observe(self, raw_means: tuple[float, ...]) -> None:
        """传入 RAW（未加权、未 z-score）的本 batch 各 component 均值。"""
        x = np.array(raw_means, dtype=float)
        if self.ema is None:
            self.ema = x.copy()
        else:
            self.ema = (1 - self.ema_alpha) * self.ema + self.ema_alpha * x
        self.step += 1
        if self.step > self.warmup_steps and self.step % self.update_every == 0:
            self._update_weights()

    def _update_weights(self) -> None:
        gap = self.ref - self.ema                      # 正 = 拖后腿
        self.weights = self.weights * np.exp(self.weight_lr * gap)
        self.weights = np.clip(self.weights, self.w_floor, self.w_cap)
        self.weights = self.weights / self.weights.sum()
        ts = datetime.utcnow().isoformat(timespec="seconds")
        msg = (
            f"[reweighter step={self.step}] "
            f"EMA={dict(zip(self.names, [round(v, 3) for v in self.ema]))} "
            f"→ w={dict(zip(self.names, [round(v, 3) for v in self.weights]))}"
        )
        print(msg)
        if self.log_path:
            with open(self.log_path, "a") as f:
                f.write(f"{self.step}\t{ts}\t"
                        f"{','.join(f'{v:.4f}' for v in self.ema)}\t"
                        f"{','.join(f'{v:.4f}' for v in self.weights)}\n")

    def get(self, name: str) -> float:
        return float(self.weights[self.names.index(name)])

4.2 改 compose_reward 内部的 reward_fn

关键改动：把原来 per-sample 一次性 计算 total reward 的循环， 拆成 三步：(a) 算每个 sample 的各 component raw 值， (b) 按 question_id 分组 z-score， (c) 让 reweighter 观察 RAW 均值并取新权重，最后加权求和。

GRPO/rewards.py — compose_reward（替换 reward_fn）
def compose_reward(
    cfg: RewardConfig,
    *,
    precedence_pairs: set[tuple[str, str]] | None = None,
    cf_cache: dict[str, dict] | None = None,
    judges: list[Callable[[str, str], float]] | None = None,
    reweighter: "HypervolumeReweighter | None" = None,
) -> Callable:
    # ... 原有的 pers_mode / ped_mode 合法性检查保留 ...

    def reward_fn(completions, **kwargs):
        prompts = kwargs.get("prompts", [None] * len(completions))
        question_ids = kwargs.get("question_id", [None] * len(completions))
        gold_plans = kwargs.get("gold_plan", [None] * len(completions))
        learner_profiles = kwargs.get("learner_profile", [None] * len(completions))

        # ---------- (a) 算 RAW 分量（不加权，不归一） ----------
        R_s, R_p, R_e, R_h = [], [], [], []
        for i, comp in enumerate(completions):
            gold = gold_plans[i] if i < len(gold_plans) else None
            qid = question_ids[i] if i < len(question_ids) else None
            profile = learner_profiles[i] if i < len(learner_profiles) else None

            R_s.append(reward_structural(comp, gold) if cfg.enable_struct else 0.0)
            R_h.append(reward_hard_gate(comp, eta=cfg.eta) if cfg.enable_hard else 0.0)

            if cfg.pers_mode == "lite":
                R_p.append(reward_personalization_lite(comp, profile))
            elif cfg.pers_mode == "counterfactual" and cf_cache and qid is not None:
                R_p.append(reward_personalization(comp, gold, cf_cache.get(qid)))
            else:
                R_p.append(0.0)

            if cfg.ped_mode == "lite":
                R_e.append(reward_pedagogy_lite(
                    comp, precedence_pairs,
                    w_precedence=cfg.w_ped_precedence,
                    w_phases=cfg.w_ped_phases))
            elif cfg.ped_mode == "full":
                lite = reward_pedagogy_lite(
                    comp, precedence_pairs,
                    w_precedence=cfg.w_ped_precedence,
                    w_phases=cfg.w_ped_phases)
                soft = reward_pedagogy_soft(comp, gold, judges) if judges else 0.0
                R_e.append(cfg.lambda_ped * lite + (1 - cfg.lambda_ped) * soft)
            else:
                R_e.append(0.0)

        # ---------- (b) 让 reweighter 观察 RAW 均值（z-score 之前） ----------
        if reweighter is not None:
            reweighter.observe((
                float(np.mean(R_s)),
                float(np.mean(R_p)),
                float(np.mean(R_e)),
            ))
            w_struct = reweighter.get("struct")
            w_pers   = reweighter.get("pers")
            w_ped    = reweighter.get("ped")
        else:
            w_struct, w_pers, w_ped = cfg.w_struct, cfg.w_pers, cfg.w_ped

        # ---------- (c) GDPO 风格 per-component z-score by question_id ----------
        def _zscore_by_group(vals, qids):
            groups = defaultdict(list)
            for idx, q in enumerate(qids):
                groups[q].append(idx)
            out = [0.0] * len(vals)
            for idxs in groups.values():
                xs = np.array([vals[i] for i in idxs], dtype=float)
                mu, sd = xs.mean(), xs.std()
                if sd > 1e-6:
                    zs = (xs - mu) / sd
                else:
                    zs = np.zeros_like(xs)
                for k, i in enumerate(idxs):
                    out[i] = float(zs[k])
            return out

        if cfg.zscore_components:
            Z_s = _zscore_by_group(R_s, question_ids)
            Z_p = _zscore_by_group(R_p, question_ids)
            Z_e = _zscore_by_group(R_e, question_ids)
        else:
            Z_s, Z_p, Z_e = R_s, R_p, R_e
        # R_hard 不做 z-score（理由见 eta 设计说明）

        # ---------- (d) 加权求和 ----------
        scores = [
            w_struct * Z_s[i] + w_pers * Z_p[i] + w_ped * Z_e[i] + R_h[i]
            for i in range(len(completions))
        ]
        if cfg.verbose:
            for i in range(len(completions)):
                print(
                    f"  [{i:02d}] raw(s,p,e)=({R_s[i]:.2f},{R_p[i]:.2f},{R_e[i]:.2f}) "
                    f"z(s,p,e)=({Z_s[i]:+.2f},{Z_p[i]:+.2f},{Z_e[i]:+.2f}) "
                    f"w=({w_struct:.2f},{w_pers:.2f},{w_ped:.2f}) "
                    f"hard={R_h[i]:+.1f} → R={scores[i]:+.3f}"
                )
        return scores

    return reward_fn

信号正常表现不对劲
reweight_log.tsv 里 w 列 缓慢漂移（每 100 步小幅变化），最终收敛到一组平稳值 w 在两个值之间反复横跳 → 调小 weight_lr

9. 调参建议

9.1 dyn_reference 怎么选

这是最重要的旋钮。选得太高 → 三项都"拖后腿"，权重全部往某个尖角倾斜；选得太低 → 早早达成 → 权重停止更新（变成固定权重）。

推荐做法：先用 SFT checkpoint 在 train split 上跑 50 个 prompt × G=8 rollout， 统计三项 raw reward 的 当前 mean + 0.10 作为 reference。例如：

SFT baseline:  R_struct=0.45  R_pers=0.20  R_ped=0.15
→ reference = (0.55, 0.30, 0.25)

这样 reweighter 一开始有动力推所有维度，但目标可达。

9.2 其他参数默认值的取舍

参数默认降低 → 调高 →
weight_lr0.3权重更稳但适应慢权重抖动大
update_every100反应快但噪声大稳但慢
ema_alpha0.10EMA 更平滑更跟最近
w_floor0.05允许某维几乎归零（危险）强制三项都有最低存在感
w_cap0.70禁止任何维一家独大允许极端倾斜（容易 hack）
warmup_steps50立刻开始调权重（数据少时不稳）固定权重跑久一些

10. 已知限制 / 进一步升级

  1. 不是真正的 HV gradient —— 这是"按差距 exp 加权"的启发式，省去了 HV 求导。 论文里 gradient-based 变体需要做 numerical HV grad，工程成本高 5-10x。 实测在 3 个 reward 的小空间里，本简化版能拿到 ≥ 80% 的提升。
  2. 跨 checkpoint 不持久化 —— reweighter 状态（EMA, weights, step）目前只在内存。 若中途 resume，weights 会从 YAML 的 init 重新开始。要彻底解决：在 save_steps 时把 reweighter.__dict__ pickle 到 output_dir/reweighter.pkl， reload 时从那读。
  3. 不和 TRL callback 集成 —— 现在权重日志只 print + tsv，没接 wandb / tensorboard。 要接入：把 _update_weights 末尾的权重写到 trainer.state.log_history。
  4. reference 是绝对值，不是 Pareto 前沿点 —— 真正的多目标做法是维护一个动态 Pareto archive。 这版简化为"固定参考点"，足够 MAP-PPL 的 3 维场景，但更高维时需要换成 archive-based。
  5. 和 segment-wise credit 正交 —— 这个改动不冲突，将来上 SPO-chain 时， reweighter 只需要在 segment-level reward 上观察均值即可。
验证通过后的下一步：开启 R_pers counterfactual（pers_mode: "counterfactual"） 和 R_ped soft judge（ped_mode: "full"），让 reweighter 在更丰富的信号上工作。 此时 reference 也要重新校准。
