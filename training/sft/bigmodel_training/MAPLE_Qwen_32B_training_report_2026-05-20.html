结果解读

Stage 1: PAD / SDP SFT

  • PAD 学习曲线整体稳定，但在 epoch 2 左右开始平台化：epoch 1.966 时 eval loss 已到 0.7216，最终停在 0.7255 附近。
  • SDP 早期下降更明显，末段稳定在 0.5178-0.5179；最终 eval token accuracy 为 0.8426。
  • 两个 SFT 阶段日志中未观察到 NaN/Inf；grad norm 与 loss 走势均属于正常可控范围。

Phase A: PAD DPO

  • DPO 将 eval reward accuracy 从 0.8364 提升到 0.8973，reward margin 从 2.539 提升到 2.723。
  • DPO train loss 不应直接与 SFT NLL 比较；本阶段更重要的信号是 chosen/rejected 分离度，而日志显示分离度在最终 eval 中继续增强。
  • pair generator 从 2433 条 source rows 产出 1198 对可用 DPO pairs。该 yield 对 structural-Jaccard proxy 来说可接受，但也说明约一半样本因为 tie 或解析失败没有进入 DPO。

Phase B: SDP on-policy SFT

  • on-policy SDP eval loss 从 0.5414 降到 0.5262，token accuracy 从 0.8393 提升到 0.8426。
  • on-policy 数据集保留 2426 / 2433 条 source rows，覆盖率接近完整训练集，因此 SDP 校正阶段数据覆盖充分。
  • 最终 on-policy SDP loss 略高于 Stage-1 SDP final loss，这是合理的：Phase B 输入来自 PAD 采样 scaffold，而不是 gold scaffold。

注意事项与下一步

Joint phase 复用了生成出的 train file 作为 valid split，因此 Phase A / Phase B 的 eval metrics 更适合作为训练稳定性与优化信号，不应直接当成 held-out dev 泛化指标。
  • 进入 GRPO 或正式汇报前，需要在 Dev 上跑 Stage-2 gate：pass@1/pass@8、GED-sim（若可用）、structural validity、agent relevance，以及小规模 LLM judge sanity check。
  • 建议为关键 checkpoint 补齐 metadata.json：data split hash、config、dev NLL、SV/AR/DC/ATR（若 evaluator 已可用）和 wall-clock runtime。
  • 当前 DPO preference proxy 是 scaffold 字段的 structural-Jaccard，不是完整 generated-plan 级别的 R_struct + R_pers + R_ped reward stack。论文或实验记录中应明确说明。
  • 日志显示 adapter 加载前执行了 embedding resize：151936 -> 151669。后续 generation/evaluation 继续使用 resize-aware loader，避免 adapter embedding shape mismatch。
