1. 为什么需要 Joint Alignment

1.1 MAPLE 的双 adapter 分解

MAPLE 把"为学习者生成多智能体计划"这个任务拆成两个串行的子任务，每个由一个 LoRA adapter 承载：

Adapter输入输出（loss target）论文符号
PAD（Personalized Agent Decomposition） query $I_q$ + 学习者 profile $I_p$ scaffold：agents $\mathcal{A}^\star$ + subtask 元信息 $\mathcal{T}^\star$ $\pi_{\text{PAD}}(\mathcal{A},\mathcal{T} \mid I_q, I_p)$
SDP（Step Decomposition Planner） query + profile + gold scaffold $(\mathcal{A}^\star, \mathcal{T}^\star)$ steps $\mathcal{S}^\star$ + execution_order $\mathcal{O}^\star$ $\pi_{\text{SDP}}(\mathcal{S},\mathcal{O} \mid I_q, I_p, \mathcal{A}^\star, \mathcal{T}^\star)$

1.2 暴露偏差 (Exposure Bias)

Stage 1 单独训练时，SDP 看到的是 gold scaffold。但推理时它消费的是 PAD 生成的 scaffold $(\mathcal{A}', \mathcal{T}') \sim \pi_{\text{PAD}}$。两个分布存在系统性 gap，这正是 Ranzato 等 (2015) 提出的经典 exposure bias 在分层生成场景下的版本：

$$ \mathbb{E}_{(\mathcal{A}^\star,\mathcal{T}^\star)\sim p_{\text{gold}}}[\mathcal{L}_{\text{SDP}}] \;\neq\; \mathbb{E}_{(\mathcal{A}',\mathcal{T}')\sim \pi_{\text{PAD}}}[\mathcal{L}_{\text{SDP}}] $$

同时 PAD 自己也不知道下游的 SDP "喜欢什么形状的 scaffold"——它只优化 NLL on gold scaffold，没有任何来自下游的反馈。这两点合起来导致 PAD↔SDP coordination bias，Joint Alignment 的目标就是同时修这两个口子。

设计哲学：narrow signal at this stage. 我们在 Joint Alignment 阶段故意只用最贴近"coordination"动机的信号：scaffold-level 结构相似度。R_pers / R_ped 这类"全套 reward"被留给 Stage 3 GRPO 独享，避免同一 reward 在 DPO 与 GRPO 里被优化两遍（消融时也能干净分离 "Joint Alignment 贡献 vs GRPO 贡献"）。

4. Phase A — PAD DPO

4.1 数学形式

对每个训练样本 $(I_q, I_p)$：

  1. 从当前 PAD 采样两个 scaffold：
    $w_1, w_2 \sim \pi_{\text{PAD}}(\cdot \mid I_q, I_p)$，温度 0.8 / top_p 0.9。
  2. 对每个 scaffold 算 scaffold-level R_struct（vs gold scaffold）：

    $$ R_{\text{struct}}(w) \;=\; \tfrac12 \cdot J_{\text{multiset}}\!\left(\text{agents}(w),\, \text{agents}^\star\right) \;+\; \tfrac12 \cdot J_{\text{set}}\!\left(\text{subtasks}(w),\, \text{subtasks}^\star\right) $$

    其中：
    • $J_{\text{multiset}}(A,B) = \sum_x \min(c_A(x),c_B(x)) \big/ \sum_x \max(c_A(x),c_B(x))$（多重集加权 Jaccard，agents 用这个因为允许重复角色）
    • $J_{\text{set}}(A,B) = |A\cap B|/|A\cup B|$（集合 Jaccard，subtask 名字用这个）
  3. 按 R_struct 排序：高者为 chosen $w$，低者为 rejected $l$。两者得分相等则丢弃。
  4. 定义 DPO log-ratio：

    $$ \Delta_{\text{PAD}}^{w,l} \;=\; \log \frac{\pi_{\text{PAD}}(w \mid I_q, I_p)}{\pi_{\text{ref}}(w \mid I_q, I_p)} - \log \frac{\pi_{\text{PAD}}(l \mid I_q, I_p)}{\pi_{\text{ref}}(l \mid I_q, I_p)} $$

  5. DPO 损失：

    $$ \boxed{\;\mathcal{L}_{\text{DPO}}^{\text{PAD}} \;=\; -\mathbb{E}_{(w,l)\sim\mathcal{D}_{\text{pref}}}\!\left[\log\sigma\!\big(\beta\,\Delta_{\text{PAD}}^{w,l}\big)\right]\;} $$

    $\beta = 0.1$，$\pi_{\text{ref}}$ 在每轮 outer iteration 开始时刷新为上一轮 PAD checkpoint。

def _multiset_jaccard(a, b) -> float:
    """Weighted Jaccard over multisets. Mirrors GRPO/plan_utils.py."""
    ca, cb = Counter(a), Counter(b)
    if not ca and not cb:
        return 1.0
    inter = sum((ca & cb).values())
    union = sum((ca | cb).values())
    return inter / union if union else 1.0


def _jaccard(a, b) -> float:
    A, B = set(a), set(b)
    if not A and not B:
        return 1.0
    return len(A & B) / len(A | B)


def scaffold_struct_score(scaffold, gold_scaffold) -> float:
    """Scaffold-only structural similarity to gold, in [0, 1]."""
    if not isinstance(scaffold, dict):
        return 0.0
    p_agents = [a.get("agent_role", "") for a in (scaffold.get("agents") or [])]
    g_agents = [a.get("agent_role", "") for a in (gold_scaffold.get("agents") or [])]
    p_subs = {s.get("name", "") for s in (scaffold.get("subtasks") or [])}
    g_subs = {s.get("name", "") for s in (gold_scaffold.get("subtasks") or [])}
    return 0.5 * _multiset_jaccard(p_agents, g_agents) + 0.5 * _jaccard(p_subs, g_subs)

def cmd_phase_b(args):
    """Plan §3.2 Phase A: DPO on PAD using scaffold-level R_struct."""
    tok, pad_model = load_base_and_adapter(
        args.base_model, args.pad_adapter, args.device, torch.bfloat16
    )
    src_rows = load_jsonl(Path(args.src).resolve())
    out_rows = []
    skipped = 0

    for chunk_start in range(0, len(src_rows), flush_every):
        chunk = src_rows[chunk_start : chunk_start + flush_every]
        jobs, prompts_data = [], []
        for r in chunk:
            inp = r["plan"]["input"]
            query, (desc, skills) = inp["query"], extract_learner(inp["learner"])
            gold_scaffold = pad_target(r["plan"]["output"])
            for pair_idx in range(args.pairs_per_query):
                jobs.append((r, pair_idx, query, desc, skills, gold_scaffold))
                # 每对样本要采 2 次 → 入队 2 份相同 prompt
                prompts_data.extend([(query, desc, skills), (query, desc, skills)])

        # 批量采样（左 padding，避免 prompt 截断）
        results = batch_sample_scaffolds(
            pad_model, tok, prompts_data,
            temperature=args.temperature, top_p=args.top_p,
            max_new=args.max_new_pad, batch_size=args.batch_size,
            phase_label="phase-b",
        )

        result_idx = 0
        for job in jobs:
            r, pair_idx, query, desc, skills, gold_scaffold = job
            candidates = []
            for _ in range(2):
                scaffold, raw_text = results[result_idx]; result_idx += 1
                score = scaffold_struct_score(scaffold, gold_scaffold)
                candidates.append((score, raw_text))
            if candidates[0][0] == candidates[1][0]:   # 平局 → 丢弃
                skipped += 1; continue
            candidates.sort(key=lambda x: -x[0])       # 降序
            chosen_text, rejected_text = candidates[0][1], candidates[1][1]
            prompt_text = render_chat(tok, [
                {"role": "system", "content": PAD_SYSTEM},
                {"role": "user",   "content": PAD_USER_TEMPLATE.format(
                    query=query, self_description=desc or "(no self-description provided)",
                    skills=json.dumps(skills, ensure_ascii=False))},
            ])
            out_rows.append({
                "prompt": prompt_text, "chosen": chosen_text, "rejected": rejected_text,
                "delta_struct": candidates[0][0] - candidates[1][0],  # 诊断用
            })

5. Phase B — SDP on-policy SFT

5.1 数学形式

把 Stage 1 SDP 的 NLL 损失从 gold scaffold 输入改成当前 PAD 采样的 scaffold 输入：

$$ \boxed{\;\mathcal{L}_{\text{SDP}}^{\text{ja}} \;=\; -\,\mathbb{E}_{(\mathcal{A}',\mathcal{T}')\sim\pi_{\text{PAD}}}\!\left[ \log \pi_{\text{SDP}}\!\big(\mathcal{S}^\star,\mathcal{O}^\star \,\big|\, I_q, I_p, \mathcal{A}', \mathcal{T}'\big) \right]\;} $$

注意：目标 $(\mathcal{S}^\star, \mathcal{O}^\star)$ 仍是 gold，只有输入变成 on-policy。这是 scheduled sampling 在分层规划任务下的实例化。

            inp = r["plan"]["input"]
            query, (desc, skills) = inp["query"], extract_learner(inp["learner"])

        for job, (scaffold, _) in zip(jobs, results):
            r, k, query, desc, skills, gold_sdp = job
            if scaffold is None or "agents" not in scaffold or "subtasks" not in scaffold:
                skipped += 1; continue

            # 把 PAD 采到的 scaffold 灌进 SDP 的 user prompt
            user_text = SDP_USER_TEMPLATE.format(
                query=query,
                self_description=desc or "(no self-description provided)",
                skills=json.dumps(skills, ensure_ascii=False),
                agents=json.dumps(scaffold["agents"],   ensure_ascii=False, indent=2),
                subtasks=json.dumps(scaffold["subtasks"], ensure_ascii=False, indent=2),
            )
            # target 仍然是 gold (S*, O*)
            assistant_text = json.dumps(gold_sdp, ensure_ascii=False, indent=2)
            out_rows.append({
                "messages": [
                    {"role": "system",    "content": SDP_SYSTEM},
                    {"role": "user",      "content": user_text},
                    {"role": "assistant", "content": assistant_text},
                ],
                "question_id": r["question_id"],
                "profile_index": r["profile_index"],
                "scaffold_sample_id": k,
            })

def load_base_and_adapter(base_model, adapter_path, device, dtype):
    """Load the base model + one PEFT adapter."""
    tok = AutoTokenizer.from_pretrained(adapter_path, trust_remote_code=True)
    if tok.pad_token is None:
        tok.pad_token = tok.eos_token

    base = AutoModelForCausalLM.from_pretrained(
        base_model, torch_dtype=dtype, device_map=device, trust_remote_code=True,
    )
    # ★ ben 5/14 踩过的坑：AutoTrain 训 LoRA 时会自动 resize tokenizer，
    #   adapter 里就带上了完整 embed_tokens（几十 MB → 1.9 GB）。
    #   加载时 base model 不知道要 resize，会形状不匹配。
    target_vocab_size = _adapter_vocab_size(adapter_path) or len(tok)
    current_vocab_size = base.get_input_embeddings().weight.shape[0]
    if current_vocab_size != target_vocab_size:
        base.resize_token_embeddings(target_vocab_size)
    model = PeftModel.from_pretrained(base, adapter_path)
    model.eval()
    return tok, model

9. 已知偏离与限制

  1. Phase A 偏好信号简化。论文/EXPERIMENT_PLAN 描述的"完整版" Phase A 应该是：PAD 采 2 scaffold → frozen SDP 各生成完整 plan → 用 $R_{\text{struct}}+R_{\text{pers}}+R_{\text{ped}}$ 全套 reward 排序。当前实现退化为 scaffold-vs-gold 结构 Jaccard，理由：
    • (a) 把 $R_{\text{pers}}/R_{\text{ped}}$ 留给 GRPO 独享，消融 RQ3 时可干净分离贡献；
    • (b) 不调 SDP forward pass，单条样本快 ~2×；
    • (c) Phase A 的动机只是修 coordination bias，结构信号最贴这个动机。
    这一偏离在 readmore_518_sft.md §"Important deviation from strict paper text" 已明文记录。
  2. valid split = train split。Phase A/B 都把生成的 train.jsonl 同时当 valid split，eval metrics 仅供训练稳定性参考。
  3. 外循环 K=1。论文允许 K ∈ {1,2}，当前只跑 K=1。K≥2 时需要把 train_joint.sh 里循环 PAD_CURRENT / SDP_CURRENT 的指针更新打通到 data 生成命令——脚本已支持，未实测。
  4. pass@1 / pass@8 gate 未自动化。Joint Alignment 出口后理论上要跑 hard gate（详见 EXPERIMENT_PLAN §3.4）才能进 GRPO，此目录暂未实现 runs/sft_gate_report.json。
  5. autotrain-advanced==0.8.36 不可用。它 pin transformers==4.48.0，加载不了 Qwen3。必须用 train_sft_lora.py / train_dpo_lora.py 直接走 TRL。
  6. DPO 数据格式。chosen/rejected 当前写成 Python list 字符串，由 train_dpo_lora.py:message_payload_to_text 解析；如改 joint_alignment_data.py 输出格式，需同步改解析器。
